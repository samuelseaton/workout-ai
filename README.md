# Workout AI

A personalized AI workout generator built with React, AWS Amplify Gen 2, and Groq. Users log in, fill out a profile and workout preferences, and receive a custom exercise plan generated by an LLM. Workouts can be saved and reviewed in a history view.

## Tech Stack

| Layer | Tech |
|---|---|
| Frontend | React 19, TypeScript, Vite |
| UI | Material UI (MUI v6) |
| Routing | React Router v7 |
| Auth | AWS Cognito (via Amplify UI Authenticator) |
| API | AWS AppSync (GraphQL) |
| Database | DynamoDB (via AppSync models) |
| AI | Groq API — `llama-3.3-70b-versatile` |
| Backend | AWS Amplify Gen 2 (CDK under the hood) |
| Hosting | AWS Amplify Hosting (CI/CD from GitHub `main`) |

## Features

- **Invite-only auth** — self-signup is disabled; accounts are admin-created only
- **Persistent user profile** — birthday, gender, height, weight are saved and pre-filled on return visits
- **Workout wizard** — step-by-step form collecting workout type, intensity, goal, target muscles, and session duration
- **AI generation** — inputs are sent to a Lambda function that calls the Groq API and returns a structured JSON workout plan
- **Workout history** — saved workouts are stored per-user in DynamoDB and viewable in a history page
- **Per-user data isolation** — AppSync ownership rules ensure users can only read/write their own records

## Architecture

```
Browser
  └── React app (Vite, hosted on Amplify)
        ├── AWS Cognito — authentication
        └── AWS AppSync (GraphQL)
              ├── UserProfile model — stores user bio data
              ├── Workout model — stores saved workout history
              └── generateWorkout query
                    └── Lambda (Node.js) → Groq API
```

## Local Development

### Prerequisites

- Node.js 20+
- AWS CLI configured (`aws configure`)
- An AWS account

### Setup

```bash
git clone https://github.com/samuelseaton/workout-ai.git
cd workout-ai
npm install
```

### Run the backend sandbox

Amplify Gen 2 deploys a personal cloud sandbox for local dev:

```bash
npx ampx sandbox
```

This provisions real AWS resources (Cognito, AppSync, Lambda) in your account and generates `amplify_outputs.json`, which the frontend reads at runtime. Keep this running while developing.

### Set the Groq secret

The Lambda needs a Groq API key. Set it once in your sandbox:

```bash
aws ssm put-parameter \
  --name "/amplify/<app-name>/<sandbox-name>/GROQ_API_KEY" \
  --value "your_groq_api_key" \
  --type SecureString \
  --region us-east-1
```

Get a key at [console.groq.com](https://console.groq.com).

### Start the frontend

```bash
npm run dev
```

App runs at `http://localhost:5173`. Auth is handled by Cognito — create a user via the AWS Cognito console (User Pools) since self-signup is disabled.

## Deployment

The app is deployed via AWS Amplify Hosting, connected to the `main` branch of this repo. Every push to `main` triggers a full build and deploy defined in `amplify.yml`:

1. **Backend** — `npx ampx pipeline-deploy` provisions/updates the Cognito, AppSync, and Lambda resources via CloudFormation
2. **Frontend** — `npm run build` outputs to `dist/`, which Amplify serves as a static site

The `GROQ_API_KEY` secret for the hosted environment is stored in AWS SSM Parameter Store at `/amplify/workoutai/main/GROQ_API_KEY`.

### Creating a user (production)

Self-signup is disabled. To add a user:

1. Go to AWS Console → Cognito → User Pools
2. Find the `workout-ai` pool
3. Users → Create user → enter email, set temporary password
4. The user will be prompted to change their password on first login
